Fragment

https://developer.android.com/guide/fragments/create

A Fragment is a piece of an application’s user interface or behavior that can be placed in an Activity. Interaction with fragments is done through FragmentManager, which can be obtained via Activity.getFragmentManager() and Fragment.getFragmentManager().

You can add your fragment to the activity’s view hierarchy either by defining the fragment in your activity’s layout file or by defining a fragment container in your activity’s layout file and then programmatically adding the fragment from within your activity.In either case, you need to add a FragmentContainerView that defines the location where the fragment should be placed within the activity’s view hierarchy.

LifeCycyle

  • onStart: when the activity becomes visible , when navigate away and come back, onStart will be called

  • onStop: when the activity becomes out of the screen. Use this to persist (permanently save) data.

  • onResume: when the activity gains focus

  • onPause: when the activity losses focus. Should keep execution fast in onRause since the next activity is not shown until it completes.

  • onCreate: only called once when the activity is first launched. Initialize variables (that initialize only once) in onCreate.

  • onDestory: It is called once when the activity is fully destroyed.

suspend

挂起。

  • 作用：函数的创建者对调用者的提醒，告诉调用者这是一个耗时函数，最好放到协程里在后台执行。会限制函数只能在协程里被调用。
  • 什么时候用？
    • 耗时的时候：I/O操作和计算工作。
      • 文件读写、网络交互、图片处理等
      • 需要等待的函数
  • 挂起后主线程就去不会执行当前代码块了，会往后走。这样主线程就不会卡。

launch {} <-> handler.post { task } +resume ( suspend挂起后回到主线程继续执行 )
