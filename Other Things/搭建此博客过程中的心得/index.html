3. 域名与GitHub Page 绑定

这里踩了不少坑。有的文章跟着走下来却没有效果。最后是跟着这篇文章的域名解析和GitHub解析部分完成的。还是要用SSH。只需要做一个A类型和一个CNAME类型两个域名解析。

需要域名和GitHub Page双向绑定。

（1）输入longqianh.github.io。由于在setting里的设置，GitHub Page会帮我重定向到我购买的longqianh.com，然后DNS服务器解析域名，将 longqianh.com 解析为我GitHub Page 的IP地址，然后得以访问。但是可能是一个IP可能对应多个web地址，直接访问IP地址是不行的。

（2）输入longqianh.com。DNS服务器解析到GitHub Page的IP地址进行访问。

CNAME

5. 一些常见问题

将本地Blog文件与remote GitHub repo链接

  • remote: Support for password authentication was removed on August 13, 2021.
  • 2021.8.13之后不能再用账号密码链接里，需要利用GitHub token
    • 登陆 GitHub –> settings –> Developer settings –> Personal access tokens –> generate new token
    • git remote set-url origin https://<token>@github.com/longqianh/blog.git
    • 这里 blog.git 是我的远程代码仓库，github.io.git 是hexo部署上去的GitHub Page的仓库，这两个仓库分离是可以实现代码同步和快速部署相互不干扰

hexo g 失败

‘YAMLException: can not read a block mapping entry; a multiline key may not be an implicit key at line 6, column 1’

解决：

title: hexo 注意冒号后面要加空格

categories 下面的’-’ 号，后面也要加空格再写字

2022.11.30

hexo g -d 报错：

这里原因是ssh连接出了问题。可以重新连一遍：

  1. local端：重新生成私有仓库密钥（注意：rsa对应着私钥，还有个ed-25519的是公钥，用在public repo）

    ssh-keygen -t rsa -C "longqianh@gmail.com"，然后一直按回车

  2. remote端：删除这个电脑之前的密钥，添加新的私钥

    • 进入 /usr/longqianh/.ssh (command+shift+. 显示隐藏文件)
    • 复制 id_rsa.pub 里的全部内容
    • 在GitHub远端粘贴

  3. ssh -T git@github.com 测试，成功

后记：其实和第一个问题是一样的，只不过解决过程中对rsa的理解加深了，ed25519对应着public repo，一开始按照github的教程[1]添加ed25519，出现了permission denied。而用rsa的原因就在于其对应private repo [2]。

参考文档：

​ [1] https://github.com/settings/keys

​ [2] https://bobcares.com/blog/permission-denied-in-publickey-ssh-github/
